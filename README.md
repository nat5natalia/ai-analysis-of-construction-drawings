# Система интеллектуального анализа строительных чертежей с помощью мультимодальных LLM

## Кратко
Прототип системы для анализа строительных чертежей (PDF, PNG, TIFF): автоматически извлекает информацию, отвечает на вопросы по чертежу, ищет по коллекции и выявляет строительные элементы (сваи, колонны и т.п.).

Команда находится на начальной стадии — проект только получен, сейчас происходит знакомство с задачей, анализ требований и планирование работ.

## Стадии и ожидания

### Стадия 0 — Инициация (текущая стадия)
- Изучение технического задания.
- Просмотр примеров строительных чертежей.
- Поиск подходящих датасетов.
- Планирование baseline экспериментов.

### Стадия 1 — Подготовка данных
- Сбор открытых наборов строительных чертежей.
- Конвертация PDF в изображения.
- Базовая обработка изображений и OCR.
- Формирование небольшой тестовой выборки.

### Стадия 2 — Baseline тестирование
- Проверка нескольких мультимодальных моделей.
- Тестирование извлечения объектов.
- Тестирование генерации текстового описания.
- Оценка качества работы моделей.

### Стадия 3 — Адаптация моделей
- При необходимости — дообучение моделей.
- Использование LoRA или PEFT.
- Повторное тестирование.

### Стадия 4 — Разработка прототипа
Создание простого web‑приложения:

- API для загрузки чертежей
- генерация текстового описания
- чат для вопросов по чертежу
- поиск по описанию
- поиск похожих чертежей

### Стадия 5 — Демонстрация и документация
- тестирование системы
- подготовка демонстрации
- оформление документации

## MVP (минимальная версия системы)

Минимальный рабочий прототип должен позволять:

- загружать строительный чертеж
- автоматически генерировать текстовое описание
- задавать вопросы по содержимому чертежа
- выполнять поиск чертежей по тексту

Дополнительные функции (например, детекция элементов) будут улучшаться постепенно.
