### **Общий роадмап проекта по неделям**

#### **Неделя 1 – Анализ, проектирование и настройка инфраструктуры**
**Общая цель:** Заложить фундамент: определить требования, выбрать технологии, настроить окружение и подготовить данные.

*   **Data Science:**
    *   Изучить статьи (ArchCAD-400K, Floorplan2Guide, ArchSeek) и существующие решения.
    *   Выбрать модели для тестирования (LLaVA, GroundingDINO, CLIP и др.).
    *   Определить метрики оценки.
    *   Найти/подготовить датасет: запросить ArchCAD-400K, собрать 100–200 чертежей с сайтов закупок, создать тестовую выборку.
    *   Согласовать с бэкендом формат хранения данных (MongoDB + вектора в ChromaDB/FAISS).
*   **Бэкенд:**
    *   Настроить проект (FastAPI, структура `backend/`).
    *   Спроектировать API (эндпоинты для загрузки, получения списка, деталей, заглушки для описания, вопросов, поиска, похожих).
    *   Настроить MongoDB (локально или в Docker), создать коллекции.
    *   Реализовать сохранение файлов и конвертацию PDF → изображения.
*   **Фронтенд:**
    *   Проанализировать требования, набросать прототипы интерфейса (Figma/бумага).
    *   Определить список страниц и компонентов.
    *   Выбрать стек (React + TypeScript, Material-UI/Tailwind, React Router).
    *   Инициализировать проект, настроить линтер, форматтер.
*   **Тестирование:**
    *   Изучить ТЗ, README, API‑спецификацию (по мере готовности).
    *   Составить тест-план, определить критические сценарии.
    *   Подготовить тестовую среду, инструменты для отслеживания багов.

#### **Неделя 2 – Разработка базового функционала и заглушек**
**Общая цель:** Создать рабочий каркас системы с мок-данными и заглушками для параллельной разработки.

*   **Data Science:**
    *   Запустить baseline‑тестирование выбранных моделей на тестовой выборке (Q&A, детекция-преобразование в граф знаний, поиск).
    *   Сравнить результаты, выбрать лучшие модели для возможного fine-tuning.
    *   Зафиксировать узкие места.
    * Реализовать функции генерации эмбеддингов:
    - get_image_embedding(image)
    - get_text_embedding(query)
    - Зафиксировать формат вектора (размерность, нормализация)
*   **Бэкенд:**
    *   Создать интерфейс для вызова DS-модулей (функции-заглушки).
    *   Доработать эндпоинт `POST /upload`: вызывать `process_drawing` (заглушку) в фоне, сохранять метаданные в БД.
    *   Реализовать эндпоинты `GET /drawings` и `GET /drawings/{id}` с реальными данными из БД.
    * Реализовать сохранение эмбеддингов:
    - При вызове process_drawing получать embedding от DS
    - Сохранять embedding в векторную БД (FAISS/ChromaDB)
    - Связывать embedding с id чертежа и метаданными в MongoDB
*   **Фронтенд:**
    *   Сверстать основные страницы с мок-данными: главная (форма загрузки, список чертежей), детальная страница (просмотрщик, блок описания, чат-заглушка), страница поиска.
    *   Настроить маршрутизацию и стейт-менеджмент (Context/Redux) с мок-данными.
*   **Тестирование:**
    *   Совместно с DS собрать набор тестовых чертежей (разные форматы, размеры, «плохие» файлы).
    *   Написать детальные тест-кейсы для всех функциональных областей (загрузка, отображение, генерация описания, чат, поиск).
    *   Начать smoke‑тестирование бэкенд-заглушек (доступность, форматы ответов).

#### **Неделя 3 – Реализация ключевых сценариев (поиск, Q&A)**
**Общая цель:** Подключить основные функции через API (пока с заглушками) и обеспечить их работу в интерфейсе.

*   **Data Science:**
    *   Если требуется fine-tuning: подготовить обучающие данные (разметка, синтетические вопросы).
    *   Запустить fine-tuning детектора (GroundingDINO) и/или LLM для Q&A (LLaVA) через LoRA/QLoRA.
    *   Оценить улучшение качества на валидационной выборке.
*   **Бэкенд:**
    *   Интегрировать векторную БД (ChromaDB/FAISS).
    *   Реализовать эндпоинты поиска: `GET /search?q=...` (через эмбеддинги текста) и `POST /similar/{id}` (по эмбеддингу изображения) – пока с заглушками.
    *   Реализовать эндпоинты `POST /drawings/{id}/describe` и `POST /drawings/{id}/ask`, вызывающие DS-заглушки.
    *   Написать интеграционные тесты для новых эндпоинтов.
*   **Фронтенд:**
    *   Подключить реальные API для загрузки, получения списка и деталей чертежа.
    *   Реализовать форму загрузки с индикацией прогресса.
    *   Интегрировать библиотеку для зума/панорамирования изображений.
    *   Начать прикручивать поиск по тексту (вызов `GET /search` и отображение результатов).
*   **Тестирование:**
    *   Провести функциональное тестирование фронтенда с мок-данными.
    *   Проверить работу новых бэкенд-эндпоинтов (заглушек) через API-тесты.
    *   Заводить баги на найденные несоответствия.

**Общая цель:** Заменить заглушки на реальные модели, обеспечить сквозную работу всех функций.

*   **Data Science:**

        *   `search_by_text` и `search_by_image`: функции для поиска.
        *   `answer_question`: мультимодальный Q&A.
    *   Протестировать время задержки между запросом и ответом системы, передать модули бэкенд-команде.
*   **Бэкенд:**

    *   Обеспечить асинхронность/фоновые задачи для долгих операций.
    *   Провести оптимизацию: индексы в БД, кэширование (Redis), профилирование запросов.
    *   Добавить детальное логирование и обработку ошибок.
*   **Фронтенд:**
    *   Реализовать вызов реальных эндпоинтов для генерации описания и чата.
    *   Доработать страницу чертежа: блоки с описанием, историей вопросов/ответов.
    *   Реализовать блок «Похожие чертежи» на детальной странице.
    *   Улучшить UX: добавить уведомления об ошибках, лоадеры.
*   **Тестирование:**
    *   Провести интеграционное тестирование всех сквозных сценариев с реальными DS-моделями.
